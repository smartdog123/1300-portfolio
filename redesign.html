Responsive Redesign

The goal of this project is to practice the workflow of redesigning a simple website. I learnt the skills necessary to analyze and identify flaws in an existing interface, create low-fidelity and high-fidelity prototypes for various screen sizes, and build a responsive website based on those prototypes.

Overview

I am redesigning the website of a Chinese food restaurant. Link to the website: chongqing house

Problem with Learnability:

  • The navigation bar is too small for audiences to read and understand.
  • Too many unnecessary text instructions on the homepage.
  • No need to show current date on the page.

Problem with Usability:

  • Static elements/Lacks reactive UI.
  • Duplicate buttons squeezed on the page bottom.
  • Hard distinguish “Cart” from other navigation bar menus.
  • “Login” is mixed with other buttons.

Problem with Memorability:

  • Too many contents, crowded, lack of visual hierarchy
  • Lots of repeated information.

Problem with Accessibility:

  • It has significant accessibility issues. WAVE report points out some errors with the current page. The main errors are form control has no or too much labels, and there are empty buttons. Other issues include redundant link, very small text, etc.

Responsiveness Summary

The responsiveness is achieved by using Bootstrap grid system and collapsable navbar. For tablet screen size, the top menu bar is collapsed, and the "Order Online" and "Order by Phone" button become vertically aligned. In terms of mobile screen, the "Chongqing House" title is made invisible, and the footer items become vertically aligned.
